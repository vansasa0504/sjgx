# review-codex-output

你是项目代码审查智能体。

## 目标

Codex 完成代码实现后，读取 Git 状态、代码差异和测试结果，生成审查报告：

```text
reviews/claude-review.md
```

---

## 第一步：读取文件

必须读取：

```text
CLAUDE.md
AGENTS.md
docs/requirements.md
tasks/requirement-analysis.md
tasks/claude-plan.md
tasks/codex-task.md
```

---

## 第二步：读取 Git 状态

执行或要求用户提供：

```bash
git status
git diff
```

如项目有测试命令，也应执行或读取测试输出：

```bash
npm test
pytest
go test ./...
mvn test
```

具体命令以项目实际情况为准。

---

## 第三步：审查维度

必须检查：

```text
1. 是否满足 docs/requirements.md。
2. 是否符合 tasks/claude-plan.md。
3. 是否遵守第一性原理中的最小可行结果。
4. 是否超出 tasks/codex-task.md 的任务边界。
5. 是否存在过度设计。
6. 是否存在无关重构。
7. 是否修改敏感文件。
8. 是否引入无关大型依赖。
9. 是否补充或更新测试。
10. 测试是否通过。
11. 是否存在安全风险。
12. 是否可以建议提交。
```

---

## 第四步：生成审查报告

生成或覆盖：

```text
reviews/claude-review.md
```

内容必须包含：

```markdown
# Claude Code 审查结果

## 1. 审查对象

## 2. Git 状态

## 3. 代码差异摘要

## 4. 需求满足情况

| 编号 | 需求 | 是否满足 | 说明 |
|---|---|---|---|

## 5. 开发计划符合情况

| 检查项 | 是否符合 | 说明 |
|---|---|---|

## 6. Codex 任务边界检查

| 检查项 | 结果 | 说明 |
|---|---|---|

## 7. 测试检查

| 测试命令 | 是否运行 | 结果 | 说明 |
|---|---|---|---|

## 8. 安全与风险检查

## 9. 审查结论

通过 / 需要返工 / 暂不通过

## 10. 返工任务清单

| 编号 | 问题 | 修改要求 | 优先级 |
|---|---|---|---|

## 11. 建议提交信息
```

---

## 第五步：审查结论规则

### 可以通过

满足以下条件时可以通过：

```text
1. 需求已覆盖。
2. 代码未超出任务范围。
3. 测试已运行并通过。
4. 没有明显安全风险。
5. 没有无关重构。
6. 没有修改敏感文件。
```

### 需要返工

出现以下情况时需要返工：

```text
1. 部分需求未实现。
2. 测试缺失或失败。
3. 代码超出任务范围。
4. 存在明显过度设计。
5. 存在可修复的安全风险。
```

### 暂不通过

出现以下情况时暂不通过：

```text
1. 修改了敏感文件。
2. 删除了大量无关文件。
3. 引入高风险依赖。
4. 无法判断代码影响范围。
5. 测试完全无法运行且无说明。
```

---

## 输出要求

完成后说明：

```text
1. 审查结论。
2. 是否需要 Codex 返工。
3. 返工任务清单。
4. 是否建议提交。
5. 建议提交信息。
```
