# 开发需求文档

> 本文件由 Claude Code 根据 `docs/technical-requirements.md` 与 `tasks/requirement-analysis.md` 生成。
> 每条需求遵循可开发、可测试、可验收、边界清晰原则。
> 编号 FR-xxx 对应需求分析中的功能需求；NFR-xxx 对应非功能需求。

## 1. 项目目标

建设金融机构统一的外部数据采集平台，实现外部数据的**扎口管理、统一接入、集中服务、全程管控**，达成：

1. **统一扎口**：全机构外部数据统一接入、管理、服务，杜绝多头接入与重复采购。
2. **高效接入**：多类型、多协议外部数据快速接入，缩短数据上线周期（新接口开发≤3个工作日，80%常规接入可视化配置）。
3. **合规管控**：数据全生命周期合规管控、质量监控、计费计量与监管统计，满足金融监管要求（等保三级、国密、金融数据安全规范）。
4. **价值释放**：建设外部数据集市，实现数据共享复用与缓存存储再利用。

**交付假设**（见需求分析 A-04）：分两期交付。
- 一期（核心）：合作方管理、外部数据接入、外部数据服务、数据目录与预览、消费方管理、数据质量管理。
- 二期（增强）：数据缓存存储再利用、计费管理、统计监管。
- 容灾、国产化、安全合规贯穿全程。

> 注：整体方向（自研建设 vs 选型评估）见 Q-01，需用户确认后冻结。

## 2. 功能需求

### 2.1 合作方管理（FR-101 ~ FR-104）

- **需求描述**：对外部数据合作方进行全生命周期、分类分级、接口与权限、服务质量监控管理。
- **输入**：合作方注册信息、资质材料、协议、接口配置、密钥、分级标签、质量指标阈值。
- **处理逻辑**：
  - 注册→资质审核→准入审批→协议管理→评级评价→退出注销状态机流转，留痕可审计。
  - 按数据类型/行业属性/合规等级/服务质量多维标签分类分级。
  - 配置接入权限、数据范围、调用频次限制；密钥安全存储（加密）。
  - 实时采集接口可用性、响应时长、数据准确率、合规性指标，超阈值预警。
- **输出**：合作方档案、分级结果、权限配置、质量监控看板与告警。
- **异常情况**：资质审核驳回、准入不通过、密钥泄露、服务质量劣化超阈值。
- **验收标准**：全生命周期流程可走通且留痕；分级与权限配置生效；质量指标实时监测并能触发预警。

### 2.2 外部数据接入（FR-201 ~ FR-205）

- **需求描述**：多类型、多协议、多格式外部数据接入，支持标准与定制两种模式，全流程管控与多种同步模式。
- **输入**：合作方数据源（API/文件/消息/DB）、协议类型、数据格式、字段映射、同步策略、调度规则。
- **处理逻辑**：
  - 协议适配层支持 HTTP/HTTPS、WebService、SFTP/FTP、Kafka、MQ、DB直连、API网关；格式适配 JSON/XML/CSV/Excel。
  - 标准接口开箱即用；定制接口可视化配置字段映射与规则，快速开发。
  - 全流程：需求申请→字段映射→规则配置→测试验证→上线审批→版本管理→下线。
  - 同步模式：增量/全量/实时推送/定时拉取，可配调度。
  - 支持断点续传。
- **输出**：接入任务、标准化落库数据、接入日志、版本记录。
- **异常情况**：协议不通、格式解析失败、字段映射缺失、同步中断、测试验证失败、审批驳回。
- **验收标准**：覆盖全部协议与格式各至少1个用例；全流程可走通；断点续传可恢复；同步模式均可配置并执行。

### 2.3 外部数据服务（FR-301 ~ FR-305）

- **需求描述**：以标准服务与定制接口对外提供数据服务，统一封装发布管控，支持高可用与权限管控，调用日志全量记录。
- **输入**：服务定义、接口参数、路由策略、限流熔断规则、权限配置、调用请求。
- **处理逻辑**：
  - 服务全生命周期：注册→接口定义→参数配置→测试→发布→版本迭代→下线归档。
  - 服务路由、负载均衡、流量控制、熔断降级、超时重试。
  - 按业务系统/角色配置调用权限，细粒度访问控制。
  - 全量记录调用请求参数、响应、耗时、状态，可追溯。
- **输出**：已发布数据服务、调用响应、调用日志。
- **异常情况**：服务超时、熔断触发、限流拒绝、鉴权失败、服务下线。
- **验收标准**：服务全生命周期可走通；路由/限流/熔断/重试策略生效；权限控制有效；调用日志可追溯。

### 2.4 数据目录访问与数据预览（FR-401 ~ FR-405）

- **需求描述**：统一外部数据目录、元信息管理、数据预览、检索与智能推荐、数据申请流程。
- **输入**：数据资产信息、元数据、分类标签、检索关键词、数据使用申请。
- **处理逻辑**：
  - 按主题/合作方/类型/场景多维分类，可视化呈现。
  - 维护字段定义、格式、更新频率、来源、合规说明、使用限制等元信息。
  - 授权范围内预览样本数据、统计信息、质量报告。
  - 关键词/标签/场景检索；基于标签+场景的规则推荐（AI推荐列为增强）。
  - 在线申请→审批→自动开通服务权限。
- **输出**：数据目录、元信息、预览结果、检索/推荐结果、申请审批记录与权限开通。
- **异常情况**：无权限预览、检索无结果、申请审批驳回。
- **验收标准**：目录多维分类可浏览；元信息完整；预览受授权控制；检索推荐可用；申请流程自动开通权限。

### 2.5 数据消费方管理（FR-501 ~ FR-505）

- **需求描述**：消费方全生命周期、分类分级、配额管理、行为审计、服务质量反馈。
- **输入**：消费方注册信息、资质、权限申请、配额配置、行为日志、反馈。
- **处理逻辑**：
  - 注册→资质审核→权限审批→配额配置→服务开通→注销。
  - 按业务条线/系统类型/合规等级分类分级，差异化管控。
  - 配额：频次/数据量/服务范围，预警与超额管控。
  - 全量记录调用/访问/操作行为，违规识别预警。
  - 收集质量与性能反馈。
- **输出**：消费方档案、配额配置、行为审计记录、告警、反馈汇总。
- **异常情况**：资质审核驳回、配额超限、违规行为识别。
- **验收标准**：全生命周期可走通；配额预警与超额拦截生效；行为审计全量可追溯；违规可识别预警。

### 2.6 数据缓存和存储再利用（FR-601 ~ FR-606，二期）

- **需求描述**：多维缓存、分级存储、脱敏加密、加工复用、外部数据集市、生命周期管理。
- **输入**：缓存策略、分层存储策略、脱敏规则、加工规则、生命周期配置。
- **处理逻辑**：
  - 接口结果/热点/全量缓存，可配时长、更新机制、失效规则。
  - 热/温/冷分层存储。
  - 存储加密(SM4)、传输加密、访问脱敏（动态+静态）。
  - 清洗、标准化、关联整合、标签加工，形成标准化资产。
  - 外部数据集市集中存储、共享复用、二次开发。
  - 留存期限、归档规则、销毁机制。
- **输出**：缓存数据、分层存储数据、加工后数据资产、集市数据、生命周期处置记录。
- **异常情况**：缓存失效、存储不足、加密/脱敏失败、生命周期到期。
- **验收标准**：缓存策略可配生效；分层存储落地；脱敏加密覆盖敏感数据；加工资产可复用；生命周期处置可执行留痕。

### 2.7 计费管理（FR-701 ~ FR-705，二期）

- **需求描述**：多维计费模型、规则配置、账单自动生成核对、费用统计分析、财务采购系统对接。
- **输入**：计费模型、计费规则、价格策略、调用/数据量统计、对接配置。
- **处理逻辑**：
  - 按调用次数/数据量/接口/套餐/时长计费。
  - 针对合作方/服务/消费方差异化规则与价格。
  - 日/月/季度自动生成合作方结算账单与消费方费用账单，核对、异议、调整审批。
  - 多维费用明细、趋势、占比统计。
  - 与财务/采购系统对接（接口规范待 Q-06 明确）。
- **输出**：计费账单、费用统计报表、对接同步结果。
- **异常情况**：计费规则冲突、账单异议、对接失败。
- **验收标准**：计费模型可配生效；账单自动生成可核对；费用统计准确；对接联调通过（待接口规范）。

### 2.8 统计监管（FR-801 ~ FR-805，二期）

- **需求描述**：全链路统计、监管报表、监管报送、合规审计追溯、可视化大屏。
- **输入**：全链路指标数据、监管报表配置、报送配置、审计查询条件。
- **处理逻辑**：
  - 接入量/调用量/传输量/缓存命中率/成功率实时统计与趋势。
  - 按监管要求生成合规/来源/个人信息报表，自定义配置与导出。
  - 对接报送系统整理/校验/加密报送（监管标准待 Q-05 明确）。
  - 全生命周期操作行为审计追溯。
  - 可视化大屏展示运行/服务/合规/成本核心信息。
- **输出**：统计指标、监管报表、报送回执、审计记录、大屏。
- **异常情况**：统计数据缺失、报表生成失败、报送校验失败。
- **验收标准**：全链路指标实时统计；报表可生成导出；审计可追溯；大屏可展示。（报送对接待规范明确）

### 2.9 数据质量管理监控（FR-901 ~ FR-906）

- **需求描述**：全链路质量监控、六维规则配置、自动校验、问题闭环、质量报告、评分评级。
- **输入**：质量规则（六维）、校验对象、问题工单、评分权重。
- **处理逻辑**：
  - 覆盖接入/传输/加工/存储/服务全环节实时监测预警处置。
  - 完整性/准确性/一致性/及时性/有效性/唯一性六维规则，可视化+自定义。
  - 对接入/加工/服务输出数据自动校验，识别异常/缺失/错误/重复。
  - 问题发现→预警→工单→整改→验证→复盘闭环。
  - 按合作方/类型/时间生成质量报告。
  - 六维加权可配评分评级（A-06）。
- **输出**：质量校验结果、告警、工单、质量报告、评分评级。
- **异常情况**：规则执行异常、质量问题闭环超时。
- **验收标准**：六维规则可配生效；自动校验识别异常；闭环流程可走通；报告可生成；评分可输出。

## 4. 数据要求

- **数据来源**：政务、征信、工商司法、运营商、互联网公开、行业垂直等外部合作方数据。
- **数据格式**：JSON、XML、CSV、Excel 等，经格式适配器转为统一内部模型。
- **数据字段**：按合作方接口定义，经字段映射落地；元信息（字段定义、格式、更新频率、来源、合规说明、使用限制）完整维护。
- **数据校验**：六维质量规则（完整性/准确性/一致性/及时性/有效性/唯一性）自动校验。
- **数据存储**：热/温/冷分级；关系型/NoSQL/时序/文件多引擎；缓存≥10TB，总存储 PB 级；存储加密(SM4)。
- **数据生命周期**：留存期限、归档规则、销毁机制可配，满足合规留存。
- **数据安全**：传输加密(TLS1.2+)、存储加密(SM4)、脱敏（掩码/替换/加密/哈希，动态+静态）、敏感数据自动识别。

## 6. 权限与安全要求

- **用户角色**：平台管理员、合作方管理员、消费方、业务用户、运维人员、开发人员；按角色差异化授权。
- **权限边界**：RBAC + ABAC，功能权限/数据权限/操作权限三维管控，粒度至字段级；按业务系统、用户角色配置服务调用权限。
- **认证方式**：MFA 多因素认证；与机构 IAM/SSO 对接单点登录；接口支持 OAuth2.0、API Key、数字证书、签名验签、时间戳防重放。
- **安全控制**：
  - 传输加密 HTTPS/TLS 1.2+；存储加密国密 SM4。
  - 数据脱敏：掩码/替换/加密/哈希，动态+静态脱敏，敏感数据自动识别。
