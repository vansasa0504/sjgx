## M6 阶段任务

### 阶段目标
48 小时稳定性测试、故障演练、升级回滚演练、文档交付、验收准备。

### 前置条件
M5 验收通过（全部性能/安全/国产化指标达标）。

### 本阶段交付物
- 48 小时连续稳定性测试（无宕机、无内存泄漏、无性能下降）。
- 故障注入演练（节点宕机、DB 主从切换、Redis 故障、MQ 故障、双活切换），验证 RPO≤5min / RTO≤30min / 单节点切换 ≤30s / 集群恢复 ≤5min。
- 灰度发布 + 滚动升级 + 回滚演练（回滚 ≤10 分钟）。
- 完整文档：系统架构文档、部署手册、运维手册、开发手册、用户操作手册。
- 验收测试报告（功能 + 性能 + 安全 + 文档）。

### 实现要求
1. 稳定性测试：持续 48h 混合负载压测，监控 JVM Heap/GC/TPS/响应时间/错误率，确认无泄漏无衰减（TPS 波动 <10%）。
2. 故障演练：用 Chaos Mesh 或脚本注入故障，验证自动切换与数据零丢失。场景：单节点宕机、DB 主从切换、Redis 故障、Kafka 故障、双活机房切换。
3. 升级演练：灰度发布 10%→50%→100%，异常自动回滚，验证回滚时间 ≤10 分钟、升级过程业务无中断。
4. 文档交付（放 docs/ 下，注意 docs/ 由 Claude Code 维护，本阶段文档放项目根 docs/delivery/ 或由 Codex 输出草稿供 Claude Code 落档；优先放 docs/delivery/）：
   - 系统架构文档（架构图、模块职责、技术选型、数据流）
   - 部署手册（环境要求、docker-compose/k8s 部署步骤、配置项说明）
   - 运维手册（监控指标、日志查看、故障排查、巡检、备份恢复）
   - 开发手册（模块结构、开发规范、新增协议/服务/规则扩展指南）
   - 用户操作手册（各功能模块操作步骤、角色权限说明）
5. 文档与实际系统功能一致，可操作。
6. 整理验收报告：功能覆盖率 100%、用例覆盖率 100%、核心通过率 100%、性能达标、安全通过、文档齐全。

### 测试要求
- 48h 稳定性：无宕机、Heap 稳定、GC 正常、TPS 波动 <10%、无内存泄漏。
- 故障演练全部场景恢复达标（切换时间/RPO/RTO/数据零丢失）。
- 升级回滚演练达标（回滚 ≤10 分钟、业务无中断）。
- 文档完整性审查（五类文档齐全且与系统一致）。

### 完成判定（本阶段验收标准）
- 48h 稳定性测试通过。
- 全部故障演练场景达标。
- 回滚演练 ≤10 分钟。
- 五类文档交付且与系统一致。
- 验收报告全部项通过（功能/性能/安全/文档）。
